# HEARTBEAT — リク 定期見守りチェック

> トリガー時にこのファイルを読んで実行する。問題なければ `HEARTBEAT_OK` をコンソール出力のみ（channelには送らない）。問題があれば詳細を channel `reply` で送信。

## チェック項目
1. **進行中タスク**: 未完了・放置タスクが無いか
2. **ブロッカー**: 承認待ち・エラーで止まっているものが無いか
3. **MCP疎通**: LINE WORKS / Discord のMCPツールが応答するか（呼ばずに存在確認）
4. **見守り対象**: SmartEarth常駐AI（姉妹ソラ等）の稼働に異常が無いか（health確認）
5. **知識ベース健全性**: Obsidian の矛盾・古い情報・孤立ページが無いか（Lint、軽量に）

## 通知ルール（厳守）
- 問題なし → **何も送らない**。`HEARTBEAT_OK` をコンソール出力のみ。
- 問題あり ＋ channel MCP 健全 → 問題詳細を channel `reply` で送信（「生存」プレフィックスは付けない）。
- 問題あり ＋ 全 channel MCP ダウン → コンソールにアラート出力のみ。

## Pro適合
- heartbeat は **60分以上間隔**。監視はシェルスクリプト中心（LLM呼ばない＝Pro cap温存）。
- 異常時のみ起動。無限リトライ禁止（genome C3）。
